Architecture-

In my full stack project I used multiple types of frontend development that each served a different purpose. The Express HTML pages relied on server side rendering and route based navigation. These pages refreshed on each request and were simpler in structure. JavaScript was used to add basic interactivity and logic to those pages. The single page application was built with Angular and ran entirely in the browser. This approach allowed dynamic updates without reloading pages. The Spa felt faster and more modern compared to Express HTML.

The backend used a NoSql MongoDB database because it works well with flexible data structures. MongoDB stores data in documents instead of tables. This made it easier to adapt changes to trip data without changing a strict schema. The database also integrates well with Node and Express. This allowed data to be passed efficiently between the backend and frontend.

Json is different from JavaScript because it is a data format not a programming language. JavaScript is used to create logic and behavior in applications. Json ties the frontend and backend together by providing a common format for Api requests and responses. The frontend sends Json data to the backend and the backend responds using the same format.

Functionality-

During the full stack process I refactored code multiple times to improve functionality and efficiency. I moved repeated logic into reusable services in Angular. I also reused user interface components to display trips and forms. The benefit of reusable UI components is consistency across the application. Reusable components also reduce duplicate code and make updates easier.

Testing-

Testing involved verifying that Api endpoints handled requests correctly. Get requests were tested to ensure data could be retrieved from the database. Put requests were tested to confirm updates were saved correctly. Security added complexity because requests required proper authentication. Invalid tokens or missing headers caused requests to fail. This showed the importance of understanding methods endpoints and security together.

I developed skills in frontend frameworks backend development and database integration. These skills make me a more marketable candidate because they reflect real world development practices.
